What: To demo my HE Hackathon work outcomes for adding a CI task to check for accessibility issues when raising a PR, or pushing new PR changes.
Why: To assist developers with possible accessibility issues, in order to identify and fix issues at an early stage - manual checks would still be required but at least some common issues may be picked up etc.
Who: Developers or anyone involved with GitHub PR work
How: Impact would be that if any accessibility issues are found, they would need to be fixed. I can only see a benefit from this as it would ensure at least some accessibility compliance is being adhered to
